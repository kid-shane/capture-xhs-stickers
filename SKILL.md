# 小红书表情包捕获

默认使用本 Skill 自带的 `scripts/run.mjs`，不启动网页。运行时使用该脚本的绝对路径；不要依赖当前工作目录。首次安装、状态异常或需要迁移平台时，读取 [references/setup.md](references/setup.md)。

## 决定动作

- 用户明确说“发到飞书”“发到收件箱”或同义表达：捕获、检查并发送到当前使用者配置的收件群。
- 用户只说“抓取”“看看”或未说明目标：捕获并检查，返回编号和结果；不要发送。
- 用户指定图片编号：只处理这些编号。没有指定时默认整组。
- 用户要求飞书云盘、微信或交互式挑图：说明自动出口仅为飞书群；不要擅自改为群发送。

## 工作流

1. 运行状态检查：

   ```bash
   node <skill-dir>/scripts/run.mjs status
   ```

   发送前需要 `feishuChat.installed=true`、`botReady=true`、`configured=true`。状态只证明本地配置存在，不证明最终发送权限；不要发送测试消息。若 `autoCleanupAvailable=false`，仍可抓取，但不能依赖自动裁水印。

2. 安全地传入链接：

   ```bash
   node <skill-dir>/scripts/run.mjs capture --stdin
   ```

   从用户消息中提取唯一的小红书 `http(s)` 链接。使用终端工具时启用交互会话，让进程等待输入；脚本会关闭终端回显。随后通过标准输入只写入该链接并追加换行。不要把链接放进 shell 命令、日志或临时配置。

3. 对每个准备发送的 `localPath` 使用平台的本地图像查看能力逐张检查；多张可并行。检查：

   - 小红书平台水印是否完全消失；
   - 底部裁切是否伤到表情主体或文字；
   - 图片是否为空白、重复、顺序异常或明显损坏。

   区分平台水印与作者署名、版权标识或台标；不要删除作者署名。裁切可疑时查看对应 `rawLocalPath`。不要生成式补画被水印覆盖的内容；无法安全处理时停止并请用户决定。平台没有本地图像查看能力时，不得越过发送门禁。

4. 只有全部选中图片通过目视检查，才发送：

   ```bash
   node <skill-dir>/scripts/run.mjs send --job <uuid> --selection all --confirmed-clean
   ```

   指定编号时使用 `--selection 1,3,5-7`。选择多张时，命令会先按顺序上传全部图片资源，全部成功后再组成一条飞书富文本消息发送；通常整组图片只产生一个消息气泡。只有富文本内容超过飞书 30 KB 上限时才自动拆成多批。单张图片继续使用飞书原生图片消息。批量消息使用稳定幂等键；不要绕过 `--confirmed-clean`。

5. 返回简短结果：笔记标题、捕获张数、实际发送张数、可见消息批次数、配置的目标群名称。上传阶段失败时说明尚未产生群消息；分批发送阶段部分成功时明确说明已发送数量和失败批次。

## 外部操作边界

- 用户明确要求发送时，视为确认接收方为其已配置的收件群、内容为本次选中图片、身份为其应用机器人。
- 用户未明确要求发送时，不调用 `send`。
- 不向其他群或个人发送，不从分享链接推断群 ID，不输出凭证或完整上游错误流。
- 不发送连接测试、欢迎语或与本次表情包无关的内容。

## 常见错误

- `XHS_LINK_TOKEN_REQUIRED`：请用户回到小红书 App，用“分享 → 复制链接”重新提供完整分享口令。
- `LARK_BOT_SCOPE_REQUIRED`：开通 `im:resource` 与 `im:message:send_as_bot`；不要让 bot 执行用户登录。
- `FEISHU_CHAT_NOT_CONFIGURED`：读取 [references/setup.md](references/setup.md)，由使用者配置自己的群。
- `FEISHU_CHAT_IMAGE_TOO_LARGE`、格式或尺寸错误：发送前停止，不要静默压缩 GIF 或改变画面。
- `LARK_BOT_NOT_IN_CHAT`：确认使用者自己的机器人仍在配置的收件群内。
